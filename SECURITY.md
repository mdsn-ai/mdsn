# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Use one of these private channels:

1. GitHub private vulnerability reporting for this repository (if enabled).
2. Directly contact the repository maintainers through private GitHub communication.

Please include:

- affected file/path,
- reproduction steps,
- impact summary,
- optional fix suggestion.

## Response Process

- Initial triage target: within 3 business days.
- Status updates are provided during investigation.
- A fix is prepared and released with a changelog note when confirmed.
