# 0.4.2 Release Notes

## Published CLI Entrypoint Fix

`create-mdsn@0.4.2` fixes the published CLI entrypoint for package-manager-driven execution.

The issue was not in scaffold generation itself. The break happened when the published bin was launched through a symlinked entrypoint, which caused the previous main-module detection to skip the real CLI startup path.

This release switches `create-mdsn` to a dedicated bin entry so the scaffold command executes consistently when launched through:

- `npm create`
- `npm exec`
- `bunx`

## Scope

- `@mdsnai/sdk` remains at `0.4.0`
- This is a focused `create-mdsn` patch release
- No starter structure changes were introduced beyond the entrypoint fix
