# 0.4.3 Release Notes

## Compatible SDK Range for 0.4 Starters

`create-mdsn@0.4.3` changes how generated starters reference `@mdsnai/sdk`.

Instead of writing `latest`, the scaffold now writes the compatible SDK range for the current minor series:

- `create-mdsn 0.4.x` generates `@mdsnai/sdk: ^0.4.0`

## Why This Changed

This keeps starter projects aligned with the intended SDK compatibility window for the `0.4` line:

- patch updates still flow automatically
- future `0.5.x` changes will not be pulled into a `0.4.x` starter by accident
- starter and SDK release planning now align by minor series
